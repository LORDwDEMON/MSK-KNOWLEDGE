# Сущность: Workplace

## Назначение сущности

Сущность Workplace предназначена для учета рабочих мест организации в системе управления охраной труда.

Рабочее место представляет собой объект производственной деятельности, на котором работник выполняет трудовую функцию и который характеризуется:

- расположением;
- выполняемыми работами;
- закрепленными должностями и работниками;
- применяемым оборудованием;
- опасными и вредными производственными факторами;
- требованиями безопасности;
- необходимыми средствами индивидуальной защиты.

Сущность является основой для процессов:

- идентификации опасностей;
- оценки профессиональных рисков;
- определения требований к обучению;
- определения норм обеспечения СИЗ;
- анализа происшествий.


## Границы сущности (Bounded Context)

### Входит:

- учет рабочих мест;
- идентификация рабочего места;
- привязка к подразделению;
- привязка к должностям;
- описание условий выполнения работ;
- учет оборудования рабочего места;
- связь с опасными факторами;
- связь с мероприятиями безопасности.


### Не входит:

- кадровые данные работников;
- трудовые отношения;
- проведение инструктажей;
- учет обучения;
- учет выдачи СИЗ;
- проведение оценки профессиональных рисков.


Эти процессы находятся в специализированных доменах:

- HR;
- Training;
- PPE;
- Risk.

## Бизнес-правила

| Код | Правило |
|-|-|
| BR-WORKPLACE-001 | Каждое рабочее место должно иметь уникальный идентификатор |
| BR-WORKPLACE-002 | Рабочее место должно относиться к конкретной организации |
| BR-WORKPLACE-003 | Рабочее место должно быть связано с подразделением |
| BR-WORKPLACE-004 | Активное рабочее место должно иметь описание условий выполнения работ |
| BR-WORKPLACE-005 | Рабочее место может иметь несколько связанных работников |
| BR-WORKPLACE-006 | Удаление рабочего места выполняется только логически |
| BR-WORKPLACE-007 | История изменений рабочего места должна сохраняться |
